Database Solution

Advantages of Solution

  • Integrity can be enforced which means data in database is always accurate such that incorrect information cannot be stored in database. In order to maintain the integrity, some integrity constraints are enforced in the database.
  • Inconsistency in the spreadsheets can be avoided i.e. When some changes are made at one cell of the spreadsheet it won’t be needed to take care of the another cell for the correct formula or data.
  • Data will be shared with multiple applicants in centralized DBMS as compared to Excel spreadsheets tool.
  • Restricting Unauthorized Access i.e. when multiple users share a database, it is likely that some users will not be authorized to access all information in the database.
  • Providing backup and Recovery i.e. a database is easy to recover from hardware or software failures.
  • Data Organization unlike in Excel spreadsheets.

Framework Being Used

  • The Framework being used is Django Framework which is built using high level Python Programming.
  • Django provides us easy access to database. We can create database objects using Django models.
  • Authenticating the user using Django Framework provides us good support. Keeps the username and passwords safe.
  • MVC (Model View Controller) is possible using Django Framework.
  • Python provides modular programming. Code readability, modifying and changes are easy and understandable
  • Django provides administrator interface to access database. So any administrator user easily change the values without depending on developer to make changes.
  • Django provides portability between different databases. For example, it is easy to switch between Oracle, PostgreSQL etc.
  • Easy to control the code between User Interface (UI) and database.
  • Open-Source need not to invest money on libraries, tools and packages.
  • Help is available on the internet and it is updated with later products and features.
  • Logging of events in log files will help us which user modified the content or data and provide right services to the customers.
  • Communities are available to provide support online. Ex. stackoverflow.com

Relational Database

Along with the Framework we need a database to store our data. For this, my proposal is to use SQLite for Development and PostgreSQL for Production.

Uses of SQLite, it is portable, Open-Source and can be used on the laptop/desktop (Support multiple operating systems Mac OS, Windows, Linux) at the time of development. We need not to have extra resources to test the application. Easy to use and help at the time of Development.

PostgreSQL for Production.

This is a an Open-Source Advanced Relational Database.

  • Well written documentation provides us easy understanding of the usage.
  • Making High-Availability, this means when primary database instance is not working we can make slave as primary will be up and running.
  • Disaster recovery is possible, we need to save our data from catastrophic and unknown disasters. To avoid these we can store data in different datacenter.
  • Sync the data can be possible to a by using right services with an less impact on services and data loss during catastrophic disasters.

Amazon Cloud Services

Use of Amazon cloud services will help us in removing the hassle and maintenance of servers and their upgradations.

Amazon provides Virtual Machines where we can add additional servers depending on load or traffic. This helps us not to worry about hardware failures. Easy to maintain, we can add additional resources like Main Memory (RAM), Harddisk can be updated depending on the usage. We pay for the resources that we use.

Since, Amazon services are online it will be easy for us to deploy the code and create Database instances at different datacenters. So they will be in sync all the time and can provide services to customers 24*7.

EC2 instances can be created with linux installed and deploy the framework and connecting to PostgreSQL provided by Amazon cloud.
